# Contributing Guidelines

We welcome contributions! Please follow these steps:

## How to Contribute
1. **Fork the repository** and clone your fork locally.
2. **Create a new branch** for your feature or bug fix:
   ```bash
   git checkout -b my-feature
   ```
3. **Make your changes**. Ensure code follows existing style and passes any linting.
4. **Write or update documentation** if you add new functionality.
5. **Run tests** (if any) and verify the project still works.
6. **Commit your changes** with a clear commit message.
7. **Push to your fork** and open a Pull Request against `main`.

## Pull Request Checklist
- [ ] Code builds without errors.
- [ ] Documentation updated where needed.
- [ ] Tests added/updated (if applicable).
- [ ] Followed the project's coding style.
- [ ] Signed the Contributor License Agreement (if required).
